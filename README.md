# portal-web-lab

## 项目简介

本仓库用于归档和展示多个独立 HTML 页面。这些页面多为前端实验、原型设计、静态演示或临时页面，彼此之间无强关联。仓库本身不绑定任何特定域名或网站，仅作为 HTML 文件的集中存放与版本管理空间。

## 目录说明

```
portal-web-lab/
├── pages/          # 存放各个独立 HTML 页面及其关联资源
├── assets/         # 公共资源（图片、样式、脚本等）
└── docs/           # 相关文档或说明文件（可选）
```

- `pages/`：每个子目录对应一个独立 HTML 页面，包含其所需的 HTML、CSS、JavaScript 文件。
- `assets/`：多个页面可能共用的资源文件，如图标、字体、通用样式库等。
- `docs/`：存放与页面归档相关的说明或设计笔记。

## 页面归档说明

- 每个页面均以独立文件夹形式存在，便于单独引用或部署。
- 页面之间无依赖关系，可独立运行。
- 归档内容可能包含实验性功能或不完整的设计，仅供存档或参考。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/portal-web-lab.git
   ```
2. 直接打开 `pages/` 下任意子目录中的 `index.html` 文件，即可在浏览器中查看对应页面。
3. 如需部署，可将 `pages/` 中的某个子目录整体上传至静态服务器。

## 维护说明

- 本仓库以存档为目的，不保证所有页面持续兼容最新浏览器版本。
- 欢迎提交 issue 或 pull request 来修正明显错误或补充说明。
- 新增页面时请遵循现有目录结构，并在 `pages/` 下创建独立子目录。
